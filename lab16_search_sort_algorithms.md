## Задание 3: Бинарный поиск — «Угадай число»

### 1. Игра «Угадай число от 1 до 100»:

**Загаданное число: 73**

| Попытка | Вопрос | Ответ |
|---------|--------|-------|
| 1 | Число больше 50? | Да |
| 2 | Число больше 75? | Нет |
| 3 | Число больше 62? | Да |
| 4 | Число больше 68? | Да |
| 5 | Число больше 71? | Да |
| 6 | Число больше 73? | Нет |
| 7 | Число больше 72? | Да |

**Ответ: 73. Потребовалось 7 попыток!**

### 2. Число от 1 до 1 000 000:

log₂(1 000 000) ≈ 20 попыток (так как 2²⁰ ≈ 1 048 576)

### 3. Почему бинарный поиск не работает на неотсортированном списке:

Бинарный поиск работает по принципу «деления пополам». Если список не отсортирован, мы не можем определить, в какой половине находится нужный элемент. Например, представим, что мы ищем число 50, спросили «больше 50?» и получили «да» — но в неотсортированном списке это ничего не значит, потому что маленькие числа могут быть вперемешку с большими. Мы не можем отбросить половину списка.

### 4. Примеры бинарного поиска в жизни:

1. **Поиск слова в словаре:** Вы открываете словарь примерно посередине, видите буквы, определяете, нужная ли это часть алфавита, и отбрасываете половину. Повторяете, пока не найдёте слово.

2. **Поиск нужной страницы в книге:** Если нужно найти главу 15 из 30 глав книги, можно открыть примерно на 15-й странице, посмотреть — и отбросить половину (до или после).

3. **Поиск контакта в телефонной книге:** Если ищете человека на букву «М», вы примерно знаете, где эта часть книги, и не листаете с начала.

## Задание 4: Сортировка пузырьком — «Шарики в воде»

### 1. Пошаговая сортировка [5, 3, 8, 1, 6, 2]:

**Исходный:** [5, 3, 8, 1, 6, 2]

**Проход 1:**
- Сравниваем 5 и 3: 5 > 3, меняем → [3, 5, 8, 1, 6, 2]
- Сравниваем 5 и 8: 5 < 8, не меняем → [3, 5, 8, 1, 6, 2]
- Сравниваем 8 и 1: 8 > 1, меняем → [3, 5, 1, 8, 6, 2]
- Сравниваем 8 и 6: 8 > 6, меняем → [3, 5, 1, 6, 8, 2]
- Сравниваем 8 и 2: 8 > 2, меняем → [3, 5, 1, 6, 2, 8]

**Проход 2:**
- [3, 5, 1, 6, 2, 8]
- 3 < 5: [3, 5, 1, 6, 2, 8]
- 5 > 1, меняем: [3, 1, 5, 6, 2, 8]
- 5 < 6: [3, 1, 5, 6, 2, 8]
- 6 > 2, меняем: [3, 1, 5, 2, 6, 8]
- 6 < 8: [3, 1, 5, 2, 6, 8]

**Проход 3:**
- [3, 1, 5, 2, 6, 8]
- 3 > 1, меняем: [1, 3, 5, 2, 6, 8]
- 3 < 5: [1, 3, 5, 2, 6, 8]
- 5 > 2, меняем: [1, 3, 2, 5, 6, 8]
- 5 < 6: [1, 3, 2, 5, 6, 8]
- 6 < 8: [1, 3, 2, 5, 6, 8]

**Проход 4:**
- [1, 3, 2, 5, 6, 8]
- 1 < 3: [1, 3, 2, 5, 6, 8]
- 3 > 2, меняем: [1, 2, 3, 5, 6, 8]
- 3 < 5: [1, 2, 3, 5, 6, 8]
- 5 < 6: [1, 2, 3, 5, 6, 8]
- 6 < 8: [1, 2, 3, 5, 6, 8]

**Проход 5:**
- [1, 2, 3, 5, 6, 8]
- 1 < 2: [1, 2, 3, 5, 6, 8]
- 2 < 3: [1, 2, 3, 5, 6, 8]
- 3 < 5: [1, 2, 3, 5, 6, 8]
- 5 < 6: [1, 2, 3, 5, 6, 8]
- 6 < 8: [1, 2, 3, 5, 6, 8]

**Результат:** [1, 2, 3, 5, 6, 8]

### 2. Количество сравнений:

Для 6 элементов: 5 + 4 + 3 + 2 + 1 = **15 сравнений**

### 3. Сравнения для больших списков:

- **20 элементов:** 19 + 18 + 17 + ... + 1 = 20 × 19 / 2 = **190 сравнений**
- **100 элементов:** 100 × 99 / 2 = **4950 сравнений**

### 4. Творческое задание — Почему «пузырёк»?

**Почему пузырёк:** Пузырёк всплывает наверх — как и самые маленькие элементы «всплывают» к началу массива при каждом проходе. Аналогия с аквариумом: тяжёлые шарики (большие числа) опускаются вниз, а лёгкие (маленькие числа) поднимаются вверх.

**Моя метафора:** Представьте, что вы стоите в очереди в столовой, где люди должны стоять по росту. Вы идёте по очереди и каждый раз меняете местами соседей, если кто-то спереди оказался ниже. Со временем самые низкие окажутся впереди — как пузырьки воздуха, всплывающие в воде.

## Задание 5: Быстрая сортировка — «Разделяй и властвуй»

### 1. Сортировка списка [7, 3, 9, 1, 5, 8, 2, 4, 6]:

**Выбираем опорный элемент: 5 (средний)**

**Разделение (Partition):**

```
Исходный: [7, 3, 9, 1, 5, 8, 2, 4, 6]
Опорный: 5

Меньшие или равны 5: [3, 1, 5, 2, 4]
Больше 5: [7, 9, 8, 6]
```

**Рекурсия для левой части [3, 1, 5, 2, 4]:**
Опорный: 3

```
Меньше 3: [1, 2]
Равны 3: [3]
Больше 3: [4, 5]

[1, 2] → уже отсортировано
[4, 5] → сортировка
```

**Рекурсия для правой части [7, 9, 8, 6]:**
Опорный: 8

```
Меньше 8: [6, 7]
Равны 8: [8]
Больше 8: [9]

[6, 7] → сортировка
```

**Сборка результата:**

```
Левая часть: [1, 2, 3, 4, 5]
Правая часть: [6, 7, 8, 9]
Итог: [1, 2, 3, 4, 5, 6, 7, 8, 9]
```

### 2. Почему быстрая сортировка быстрее пузырька:

**Простым языком:** представьте, что вам нужно найти самого высокого ученика в школе.

- **Пузырёк:** Вы проходите по всей школе, спрашиваете каждого, запоминаете самого высокого. Потом снова проходите по всей школе для второго места, и так далее. Много лишних сравнений!

- **Быстрая сортировка:** Вы выбираете одного ученика как «ориентир» и делите всех на тех, кто ниже и выше. Потом работаете с маленькими группами отдельно. Это как если бы вы сначала определили, в каком классе примерно находится самый высокий, и не проверяли остальные классы.

**Сложность:** O(n log n) vs O(n²) — разница огромная при больших данных!

### 3. Творческое задание — «Разделяй и властвуй» в быту:

**Поиск нужной страницы в книге:**
1. Открываю книгу примерно посередине
2. Смотрю, какая глава там начинается
3. Решаю, нужен ли мне материал до или после
4. Открываю новую середину выбранной части
5. Повторяю, пока не найду нужную страницу

Это как искать нужную страницу в словаре — никто не листает его с первой страницы!

## Задание 6: Сравнение алгоритмов

| Алгоритм | Сложность | Для 100 элементов | Для 100 000 элементов | Когда использовать |
|----------|-----------|-------------------|----------------------|-------------------|
| Линейный поиск | O(n) | ~100 операций | ~100 000 операций | Неотсортированные данные, небольшие списки |
| Бинарный поиск | O(log n) | ~7 операций | ~17 операций | Отсортированные данные, большие списки |
| Пузырьковая сортировка | O(n²) | ~10 000 операций | ~10 000 000 000 операций | Образование, очень маленькие списки (<20 элементов) |
| Быстрая сортировка | O(n log n) | ~664 операции | ~1 664 000 операций | Большие списки, реальные приложения |

### Вопросы для размышления:

**1. Для скольких элементов можно использовать пузырёк (< 1 секунды)?**

Пузырёк делает n × (n-1) / 2 сравнений. При 10 000 сравнений в секунду:
- Для < 1 секунды: n² / 2 < 10 000 → n < 141

**Пузырёк разумен для списков до ~100-200 элементов.**

**2. Что будет, если применить бинарный поиск к неотсортированному списку?**

Результат будет **непредсказуемым** — алгоритм может вернуть неправильный ответ или не найти существующий элемент. Бинарный поиск работает только когда элементы отсортированы, потому что он опирается на знание «все элементы слева меньше, справа больше».

**3. Какую сортировку выбрать для 1 000 000 элементов?**

**Быструю сортировку (QuickSort)** или **Timsort** (используемый в Python). Это потому что:
- O(n log n) — достаточно быстро
- Хорошо работает с большими данными
- Не использует много дополнительной памяти

**4. Какую сортировку использует Python и почему?**

Python использует **Timsort** — гибридный алгоритм, основанный на сортировке слиянием и вставкой.

**Почему лучше самодельных:**
- Оптимизирован для реальных данных (часто содержат частично отсортированные участки)
- Гарантирует O(n log n) в худшем случае
- Очень эффективен на практике
- Прошёл тысячи тестов и доказал свою надёжность

## Задание 7: Творческий проект

### Программа «Библиотекарь-бот»

### Что делает программа:

Приложение для школьной библиотеки, которое помогает:
1. Сортировать книги по алфавиту (по автору или названию)
2. Быстро находить нужную книгу
3. Вести учёт выданных книг

### Какие алгоритмы используются:

1. **Быстрая сортировка (QuickSort)** — для сортировки каталога книг по названию или автору. Выбираем первую букву как опорную точку и делим книги на группы.

2. **Бинарный поиск** — для нахождения книги в отсортированном каталоге. Библиотекарь вводит название, и система быстро находит книгу.

### Примерный интерфейс:

```
=== БИБЛИОТЕКАРЬ-БОТ ===
------------------------

Меню:
1. Добавить книгу
2. Показать все книги
3. Найти книгу
4. Сортировать каталог
5. Выйти

> Введите название книги: Гарри Поттер

Поиск...
Найдено: "Гарри Поттер и философский камень"
         Автор: Дж. К. Роулинг
         Год: 1997
         Место на полке: 3А
         
         Найдено: "Гарри Поттер и тайная комната"
         ...
         
Найдено за 3 сравнения из 150 книг!
```

### Почему выбраны именно эти алгоритмы:

1. **Быстрая сортировка:**
   - Библиотекарь часто добавляет новые книги
   - Нужно поддерживать порядок на полках
   - Быстрая сортировка эффективна для больших каталогов
   - Не использует много памяти (важно для старых компьютеров в библиотеках)

2. **Бинарный поиск:**
   - Каталог всегда отсортирован
   - Ученики часто ищут конкретные книги
   - 150 книг → максимум 7 сравнений вместо 150!
   - Экономит время библиотекаря

**Альтернатива:** Можно использовать пузырёк, но это будет в 100 раз медленнее для большой библиотеки!
